## Background

스트레스 테스트는 웹 서비스의 안정성을 보장하기 위해 필수적이지만 소규모 팀의 경우 테스트의 복잡성과 비용 때문에 충분한 테스트를 수행하기 어렵습니다.
이 프로젝트는 자동화되고, 빠르며, 사용이 쉽고, 비용 효율적인 테스트 환경을 제공하여 이러한 스트레스 테스트의 장벽을 낮추는 것을 목표로 진행했습니다.

## What I Built

EzStress는 LLM을 사용해 API 트래픽을 추정하고 현실적인 사용자 패턴을 기반으로 스트레스 테스트 시나리오를 자동 생성합니다.
또한 AWS Fargate 기반으로 확장 가능한 테스트와 실시간 모니터링을 지원하며 서비스 최적화를 위한 인사이트를 제공합니다.

## What I Did

- 6인 팀에서 TypeScript와 React를 사용해 프론트엔드를 설계·개발
- 정보 구조 설계, 와이어프레임 및 UI를 Figma로 제작하고 사용성 중심으로 개선
- 비용 조회, 한도 설정 등 기본 user api 연결
- 서버에 요청 부하를 보내고 나온 결과를 GET api로 받아서 결과 차트/그래프로 시각화
  - 타임아웃 비율
  - 시간별·사용자수별 응답시간
  - 상태 코드 비율
  - 시간별 200·500 등 응답 상태
  - 사용자가 가장 많이 들어갈 것으로 예상되는 top endpoint
  - 발생하는 에러 메세지 nlp 분석
- AWS S3를 사용해 배포
<br/>
## Result

EzStress는 다음 두 가지 장점을 갖고 있습니다.
- 비용 효율: 100명 기준으로도 테스트 비용이 0.10달러 이하로, 별도의 라이선스나 고가의 인프라 없이도 충분한 부하 테스트를 수행할 수 있습니다.
- 사용성: 복잡한 설정이나 스크립트 작성 없이도, 사용자는 몇 가지 테스트 조건만 입력하면 됩니다. 시스템이 자동으로 테스트를 실행하고, 대부분 몇 시간 내에 결과를 확인할 수 있습니다.
